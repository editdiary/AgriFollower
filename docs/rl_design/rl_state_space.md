# 📝 [설계 문서] 강화학습 기반 추종 로봇의 상태 공간(State Space) 및 센서 퓨전 파이프라인 설계

## 1. 개요
본 문서는 수확 보조 자율 운반 로봇의 강화학습(RL) 에 에이전트가 환경을 직관적이고 효율적으로 이해할 수 있도록 복잡한 원시 센서 데이터를 고도로 압축하고 정제하는 상태 공간(State Space) 설계안을 다룬다. 현장 노이즈를 고려하여 센서별 역할을 명확히 분리하고, 전처리 파이프라인을 구축하여 학습 속도와 심투리얼(Sim-to-Real) 강건성을 극대화하는 것을 목적으로 한다.

## 2. 센서 데이터 가공 및 차원 압축 전략
### 2.1. RGB-D 비전 (타겟): 특정 마커 기반 다차원 타겟 추종 특징 (4차원)
픽셀 단위의 고차원 이미지와 Depth 맵을 직접 사용하는 대신, 경량화된 비전 파이프라인을 거쳐 단 하나의 신뢰할 수 있는 타겟 정보로 압축하여 1D 벡터에 포함한다. 기존의 Bounding Box 크기를 통한 거리 간접 유추 방식을 폐기하고, 물리적인 실측 거리를 직접 사용한다.
- 가공 방식: 작업자에게 부착된 특정 마커(예: AprilTag)를 검출하고, 해당 픽셀 영역의 중앙 Depth 값을 추출하여 카메라 좌표계 기준의 상대 좌표로 변환한다.
- 추출 특징 (4차원):
    - $x_{norm}, y_{norm}$: 카메라 화면 중심 기준 타겟의 정규화된 2D 픽셀 위치 (시각적 조향 기준)
    - $d_t, \theta_t$: 카메라 렌즈 기준 타겟과의 실제 유클리디안 거리 및 상대 각도 (물리적 속도 제어 기준)

### 2.2. RGB-D 비전 (장애물): 하단부 사각지대 방어 (3차원)
로봇의 최상단에 부착된 2D LiDAR가 스캔하지 못하는 'LiDAR 평면 하단 ~ 지면' 사이의 3차원 사각지대(예: 바닥 파이프, 수확 대차 하단부)를 인지하기 위해 Depth 데이터를 가상의 하단 범퍼로 활용한다.
- 가공 방식 (가상 뎁스 범퍼): Depth 맵 전체가 아닌, 로봇 하단 높이에 해당하는 '중·하단 픽셀 영역'만 마스킹하여 관심 영역(ROI)으로 설정한다. 이를 좌/중/우 3개 구역으로 분할하여 최소 깊이 값을 추출한다. 'LiDAR 평면 하단'의 경우 물리적 offset(Extrinsic)과 카메라 내부 파라미터(Intrinsic)를 활용해 투영하여 계산한다.
- 추출 특징 (3차원):
    - $d_{depth\_left\_low}$: 좌측 하단 3D 사각지대 최단 거리
    - $d_{depth\_center\_low}$: 전방 하단 3D 사각지대 최단 거리
    - $d_{depth\_right\_low}$: 우측 하단 3D 사각지대 최단 거리

### 2.3. 2D LiDAR: 구역 분할(Sector Pooling) 및 노이즈 필터링
방대한 원시 거리 레이저 데이터(360° 스캔, 0.8°/샘플)를 그대로 입력할 경우의 학습 저하를 막기 위해 기하학적 안전 여백 특징으로 압축한다.
- 가공 방식 (부채꼴 구역 분할): 로봇의 360도 시야각을 6개의 핵심 부채꼴 구역(좌전방, 정면, 우전방, 우하방, 후면, 좌하방)으로 분할한다.
- 노이즈 필터링 전략: 각 구역의 생(Raw) 최솟값을 그대로 쓰면 먼지나 반사광 노이즈로 인해 급제동이 걸릴 수 있으므로 아래 전처리를 필수 적용한다.
    - 백분위수(Percentile) 필터: 각 구역 내 하위 3~5% 수치들을 최솟값 경계로 간주하여 단일 점(Outlier) 노이즈 무시.
    - 결측치 클리핑(Clipping): 센서 측정 범위를 벗어나는 NaN이나 Inf 값을 최대 감지 거리(Max Range)로 강제 치환.
    - 지수 이동 평균(EMA): 프레임 간 거리 값이 급격하게 튀는 현상을 방지하기 위해 시간축 평활화 적용.
- 추출 특징 (6차원): 전처리 필터가 적용된 각 구역별 '최종 안전 마진 거리 값'.

### 2.4. IMU / Odometry: 로봇 동역학 상태
로봇이 자신의 주행 관성과 미끄러짐을 스스로 인지하여 급발진/급제동을 방지할 수 있도록 필터링된 속도 성분을 제공한다.
- 추출 특징 (3차원):
    - $v_x$: 현재 로컬 기준 전/후진 선속도
    - $v_y$: 현재 로컬 기준 좌/우 평행 이동 선속도 (매카넘 휠 특성 반영)
    - $\omega$: 현재 제자리 회전 각속도

### 2.5. 센서 기하 파라미터 (Intrinsic / Extrinsic) 획득 및 활용
§2.1의 역투영과 §2.2의 "LiDAR 평면 하단" 투영에 필요한 카메라 내부 파라미터(Intrinsic)와 Cam–LiDAR 상대 변환(Extrinsic)은 시뮬레이션에서 **이미 ground truth로 존재**한다 — 캘리브레이션 불필요. (구체적 수치·출처·주의사항의 단일 출처는 `docs/environment.md` "카메라 Intrinsic & Cam–LiDAR Extrinsic" 절.)
- 획득 방법 (RL env `reset()`에서 1회 조회 후 캐시):
    - Intrinsic: `/depth_cam/camera_info`의 K 행렬 ($f_x = f_y \approx 554.26$, $c_x = 320$, $c_y = 200$, 왜곡 0)
    - Extrinsic: TF `lookup_transform('camera_link0', 'lidar_frame')` — 평행이동 $[-0.0840, \approx 0, +0.0592]$ m, 회전 없음 (fixed joint)
- §2.1 활용 (마커 역투영): 마커 픽셀 $(u, v)$ + 해당 픽셀 Depth $Z$ 에서
    $$X = \frac{(u - c_x) Z}{f_x}, \quad Y = \frac{(v - c_y) Z}{f_y}, \quad d_t = \sqrt{X^2 + Y^2 + Z^2}, \quad \theta_t = \arctan\left(\frac{X}{Z}\right)$$
    정규화 픽셀 위치도 동일 파라미터 사용: $x_{norm} = (u - c_x)/c_x$, $y_{norm} = (v - c_y)/c_y$.
- §2.2 활용 (가상 뎁스 범퍼 ROI): LiDAR 스캔 평면은 카메라 광학중심보다 $\Delta z = +0.0592$ m 위에 있으므로, 깊이 $Z$인 픽셀이 LiDAR 평면 아래(사각지대)일 조건은
    $$v > c_y - f_y \cdot \frac{\Delta z}{Z}$$
    (예: $Z = 1$ m에서 $v > 167$). ROI 경계가 깊이 의존적이므로, 구현 단순화 시에는 보수적으로 화면 하반($v > c_y$)을 고정 ROI로 써도 무방하다 — LiDAR 평면보다 항상 아래쪽만 포함되어 안전 측.
- Sim-to-Real: 실기에서는 `camera_info`/TF 자리에 실측 캘리브레이션 결과(체커보드 intrinsic, 타겟 기반 Cam–LiDAR extrinsic)를 **동일 인터페이스로 주입**하면 파이프라인 코드는 불변이다.
- ⚠️ `camera_link0`은 optical frame이 아닌 X-forward body 좌표계 — 픽셀↔3D 변환 시 광학 관례 회전 필요 (상세는 `docs/environment.md`).

## 3. 최종 상태 공간 (State Space) 정의
다층 퍼셉트론(MLP)의 고정 크기 입력을 만족하고 정적 프레임에서는 알 수 없는 타겟의 이동 속도 및 방향성을 유추할 수 있도록 시계열 상태 누적(Frame Stacking) 구조를 취한다.
- 단일 프레임 상태 벡터 ($s_t$):
    $$\text{RGB 마커 특징(4D)} + \text{하단 depth 범퍼(3D)} + \text{LiDAR 구역별 최솟값(6D)} + \text{로봇 속도 벡터(3D)} = \mathbf{16\text{차원}}$$
- 최종 입력 상태 공간 ($S_t$):
    최근 3개 프레임의 데이터를 하나로 이어 붙여 최종 입력을 구성한다.
    $$S_t = [s_t, s_{t-1}, s_{t-2}] = \mathbf{\text{총 } 48\text{차원 1D 벡터}}$$

## 4. 주요 우려 사항 및 접근 방향성
### ⚠️ 우려 1: 다중 작업자 간섭 및 비전 오탐지 문제
- 현상: 이동 경로 상에 다른 작업자가 나타나거나 일시적으로 카메라 노이즈가 발생할 경우, 추종 대상이 바뀌거나 로봇이 비정상적으로 거동할 위험이 있음.
- 해결 방향성 (Tracking-by-Detection):
    - YOLO와 같은 범용 인물 탐지 대신, 고유 번호가 지정된 특정 마커(AprilTag 등)를 주 타겟으로 한정하여 타겟 식별 오류를 근본적으로 차단함.
    - 비전 처리단에 칼만 필터(Kalman Filter) 기반의 객체 추적(SORT 계열)을 결합하여, 순간적으로 마커를 놓치더라도 이전 프레임의 위치와 속도로 타겟의 궤적을 예측·유지함으로써 강화학습 에이전트에게는 항상 끊김 없고 정제된 단 하나의 타겟 좌표만 공급함.
### ⚠️ 우려 2: 2D LiDAR의 태생적 한계 (Negative Obstacle 및 3차원 사각지대)
- 현상: 철골 구조물의 모서리, 얇은 파이프 다리 등 2D LiDAR의 얇은 레이저 평면 높이와 맞지 않는 장애물은 로봇이 '빈 공간'으로 오인하여 충돌할 수 있음.
- 해결 방향성:
    - 1차 방어 (현행 설계): 시뮬·실물 모두 RGB-D를 탑재했으므로, §2.2의 가상 뎁스 범퍼(하단 ROI 좌/중/우 최단 거리)를 상태에 포함하고 보상의 전방 안전 페널티에 융합($d_{front\_merged}$, `rl_reward_function.md` §3.2)하여 하단 사각지대 충돌을 방어함.
    - 고도화 (필요 시): Depth 데이터를 기반으로 3D 포인트 클라우드를 생성한 뒤 이를 가상의 2D 레이저 스캔으로 투영(depthimage_to_laserscan)하여 실제 2D LiDAR 토픽과 융합(Sensor Fusion)하는 결정적(deterministic) 안전 레이어를 추가로 보강함.

# 📝 [설계 문서] 강화학습 기반 추종 로봇의 상태 공간(State Space) 및 센서 퓨전 파이프라인 설계

## 1. 개요
본 문서는 수확 보조 자율 운반 로봇의 강화학습(RL) 에 에이전트가 환경을 직관적이고 효율적으로 이해할 수 있도록 복잡한 원시 센서 데이터를 고도로 압축하고 정제하는 상태 공간(State Space) 설계안을 다룬다. 현장 노이즈와 하드웨어 제약을 고려하여 센서별 역할을 명확히 분리하고, 전처리 파이프라인을 구축하여 학습 속도와 심투리얼(Sim-to-Real) 강건성을 극대화하는 것을 목적으로 한다.

## 2. 센서 데이터 가공 및 차원 압축 전략
### 2.1. RGB 비전: 특정 마커 기반 타겟 추종
픽셀 단위의 고차원 이미지를 직접 사용하는 대신, 경량화된 비전 파이프라인을 거쳐 단 하나의 신뢰할 수 있는 타겟 정보로 압축하여 1D 벡터에 포함한다.
- 가공 방식: 작업자에게 부착된 특정 마커(예: AprilTag)를 검출하여 이미지 평면 상의 Bounding Box 정보를 추출한다.
- 추출 특징 (4차원):
    - $x_{center}, y_{center}$: 이미지 중심 기준 마커의 상대적 상/하/좌/우 위치 (로봇의 조향 및 평행 이동의 기준)
    - $Width, Height$: 마커의 가로/세로 크기 (Depth 센서가 없는 환경에서 타겟과의 상대적 거리를 유추하는 간접 지표로 활용)

### 2.2. 2D LiDAR: 구역 분할(Sector Pooling) 및 노이즈 필터링
270개의 방대한 원시 거리 레이저 데이터를 그대로 입력할 경우의 학습 저하를 막기 위해 기하학적 안전 여백 특징으로 압축한다.
- 가공 방식 (부채꼴 구역 분할): 로봇의 270도 시야각을 5개 혹은 7개의 핵심 부채꼴 구역(예: 좌측, 좌전방, 정면, 우전방, 우측)으로 분할한다.
- 노이즈 필터링 전략: 각 구역의 생(Raw) 최솟값을 그대로 쓰면 먼지나 반사광 노이즈로 인해 급제동이 걸릴 수 있으므로 아래 전처리를 필수 적용한다.
    - 백분위수(Percentile) 필터: 각 구역 내 하위 3~5% 수치들을 최솟값 경계로 간주하여 단일 점(Outlier) 노이즈 무시.
    - 결측치 클리핑(Clipping): 센서 측정 범위를 벗어나는 NaN이나 Inf 값을 최대 감지 거리(Max Range)로 강제 치환.
    - 지수 이동 평균(EMA): 프레임 간 거리 값이 급격하게 튀는 현상을 방지하기 위해 시간축 평활화 적용.
- 추출 특징 (5~7차원): 전처리 필터가 적용된 각 구역별 '최종 안전 마진 거리 값'.

### 2.3. IMU / Odometry: 로봇 동역학 상태
로봇이 자신의 주행 관성과 미끄러짐을 스스로 인지하여 급발진/급제동을 방지할 수 있도록 필터링된 속도 성분을 제공한다.
- 추출 특징 (3차원):
    - $v_x$: 현재 로컬 기준 전/후진 선속도
    - $v_y$: 현재 로컬 기준 좌/우 평행 이동 선속도 (매카넘 휠 특성 반영)
    - $\omega$: 현재 제자리 회전 각속도

## 3. 최종 상태 공간 (State Space) 정의
다층 퍼셉트론(MLP)의 고정 크기 입력을 만족하고 정적 프레임에서는 알 수 없는 타겟의 이동 속도 및 방향성을 유추할 수 있도록 시계열 상태 누적(Frame Stacking) 구조를 취한다.
- 단일 프레임 상태 벡터 ($s_t$):
    $$\text{RGB 마커 특징(4차원)} + \text{LiDAR 구역별 최솟값(5~7차원)} + \text{로봇 속도 벡터(3차원)} = \mathbf{12 \sim 14\text{차원}}$$
- 최종 입력 상태 공간 ($S_t$):
    최근 3개 프레임의 데이터를 하나로 이어 붙여 최종 입력을 구성한다.
    $$S_t = [s_t, s_{t-1}, s_{t-2}] = \mathbf{\text{총 } 36 \sim 42\text{차원 1D 벡터}}$$

## 4. 주요 우려 사항 및 접근 방향성
### ⚠️ 우려 1: 다중 작업자 간섭 및 비전 오탐지 문제
- 현상: 이동 경로 상에 다른 작업자가 나타나거나 일시적으로 카메라 노이즈가 발생할 경우, 추종 대상이 바뀌거나 로봇이 비정상적으로 거동할 위험이 있음.
- 해결 방향성 (Tracking-by-Detection):
    - YOLO와 같은 범용 인물 탐지 대신, 고유 번호가 지정된 특정 마커(AprilTag 등)를 주 타겟으로 한정하여 타겟 식별 오류를 근본적으로 차단함.
    - 비전 처리단에 칼만 필터(Kalman Filter) 기반의 객체 추적(SORT 계열)을 결합하여, 순간적으로 마커를 놓치더라도 이전 프레임의 위치와 속도로 타겟의 궤적을 예측·유지함으로써 강화학습 에이전트에게는 항상 끊김 없고 정제된 단 하나의 타겟 좌표만 공급함.
### ⚠️ 우려 2: 2D LiDAR의 태생적 한계 (Negative Obstacle 및 3차원 사각지대)
- 현상: 철골 구조물의 모서리, 얇은 파이프 다리 등 2D LiDAR의 얇은 레이저 평면 높이와 맞지 않는 장애물은 로봇이 '빈 공간'으로 오인하여 충돌할 수 있음.
- 해결 방향성:
    - 단기 (시뮬레이션 단계): 초기 개발 단계의 복잡성을 낮추기 위해, 현재 일반 RGB 기반 환경에서는 해당 문제를 일시 유예하고 기본적인 마커 추종 및 고랑(벽) 회피 메커니즘 학습에 집중함.
    - 장기 (실물 포팅 및 고도화 단계): 하드웨어 사양이 RGB-D(깊이 카메라)로 확장 가능한 시점이 오면, Depth 데이터를 기반으로 3D 포인트 클라우드를 생성한 뒤 이를 가상의 2D 레이저 스캔으로 투영(depthimage_to_laserscan)하여 실제 2D LiDAR 토픽과 융합(Sensor Fusion)하는 방향으로 사각지대를 방어함.

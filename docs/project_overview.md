# 📘 [프로젝트 종합] 온실 작업자 추종 강화학습 — 설계부터 학습 실험까지

> **이 문서의 용도:** 프로젝트 전체(배경 → 시스템 → MDP 설계 → 구현 → 학습 실험 이력 → 평가 → 결론)를
> 한 번에 파악하기 위한 종합 문서이자 **보고서 초안**. 구체 수치·수식의 단일 출처는
> `rl_design/` 세부 노트, 코드 검증·튜닝은 `rl_code_guide.md`, 실행 명령어는 `runbook.md`이며,
> 이 문서는 그것들을 **하나의 서사로 연결**한다.
> 조정 차수(1~7차)의 연대기는 이 문서 §5가 단일 출처다.

## 1. 배경 및 목표

**AgriFollower**는 온실에서 수확 작업자를 따라다니며 수확물을 운반해 작업 피로를 줄이는
수확 보조 자율 운반 로봇 프로젝트다. 작업자가 수확한 과일을 팔만 뻗어 로봇 적재함에
바로 담을 수 있어야 하므로, 핵심 과제는 단순 통로 주행/내비게이션이 아니라 **타겟(작업자) 추종** —
좁은 통로(좌우 여유 ~10cm)에서 작업자와 근접 거리(0.87m, = 앞범퍼↔신체외곽 0.50m)를 유지하면서 장애물에 맞춰
최적 자세를 잡는 주행 정책이다.

본 워크스페이스는 그중 **시뮬레이션 + 강화학습 파트**로, 실 하드웨어 없이
Ignition Gazebo 온실에서 정책을 학습하고 검증한다.

**당초 계획 대비 실제 범위** — 착수 시점 계획서(`rl_design/0_project_proposal.md`)는
"2주 내 Sim-to-Real 실물 포팅 검증까지"를 범위로 했으나, 실제로는 다음으로 조정되었다
(수업 프로젝트 범위 확정):

| 항목 | 당초 계획 | 실제 수행 |
|---|---|---|
| 시뮬 환경 구축 (온실·센서·RL 브리지) | ✓ | ✅ 완료 |
| 학습 시나리오 | 5종 (정속·Stop&Go·후진·지그재그·U턴) | 시나리오 1(정속 왕복)만 구현 — 단, 왕복 복귀 구간에서 후진 추종(시나리오 3의 핵심 거동)이 자연 포함됨 |
| 알고리즘 | SAC(1순위) + PPO(베이스라인) 비교 | SAC 3개 런 완료/진행 + PPO _예정_ |
| Sim-to-Real (Jetson 실물 포팅) | 기초 검증까지 | **범위 외** — 설계상 교체 지점만 정의 (§7) |

## 2. 시스템 구성

| 항목 | 내용 |
|------|------|
| 시뮬레이터 | **Ignition Gazebo Fortress 6.x** · 렌더 `ogre2`(GPU) — 센서 토픽이 실물과 동일 구조라 채택 (proposal §4.1) |
| 로봇 | ROSOrin Mecanum (Hiwonder, 4 매카넘 휠, 3-DOF 홀로노믹) — 온실 스케일에 맞춘 균일 1.83× 포크 (`greenhouse_sim/urdf/`) |

| 실행 환경 | Docker `nvidia-egl-desktop-ros2:humble` (GPU EGL 렌더, noVNC) · Ubuntu 22.04 · ROS 2 Humble |

## 3. RL 문제 정의 (MDP)

> 수치·수식·전처리의 단일 출처: 상태 → [`rl_design/rl_state_space.md`](rl_design/rl_state_space.md) ·
> 보상/종료 → [`rl_design/rl_reward_function.md`](rl_design/rl_reward_function.md) ·
> 시나리오 → [`rl_design/rl_train_scenarios.md`](rl_design/rl_train_scenarios.md)

- **상태 (48차원):** 센서 원시 데이터를 기하학적 특징으로 압축한 단일 프레임 16차원
  — 타겟 특징 `[x_norm, y_norm, d_t, θ_t]`(4) + RGB-D 하단 뎁스 범퍼 좌/중/우(3, 2D LiDAR
  사각지대 보완) + LiDAR 6구역 최솟값(6) + 로봇 속도 `[vx, vy, ω]`(3) — 을 최근 3프레임
  스택해 48차원. 프레임 스택으로 타겟 이동 방향·로봇 관성을 동역학적으로 유추 가능.
- **행동 (3-DOF 연속):** `[ax, ay, aω] ∈ [-1,1]³` → `vx, vy`(±0.5 m/s), `ω`(±1.0 rad/s).
  매카넘 홀로노믹 — 헤딩 고정 상태의 게걸음(vy) 중앙 보정이 설계 의도.
- **보상 (6항):** 추종 가우시안(d_opt=0.87m) + 접근 shaping + 전방/측면 안전 페널티(3D 퓨전,
  클리핑 2차) + 자세(yaw 정렬·중앙 유지, 게이트 분리 활성화) + 행동 평활 + 타겟 주시.
  통로 내부/외부를 LiDAR 좌우 거리로 판단해 자세 항을 연속 게이트로 전환.
- **종료 (5조건):** 성공(+100, max_steps=1500 생존 = 통로 왕복 완주와 등가) /
  환경 충돌(-30) / 타겟 충돌(-100, d_t<0.62m) / 타겟 이탈(-50, d_t>3.0m 또는 15프레임 미검출) /
  정체(-50).
- **도메인 랜덤화:** 타겟 특징 가우시안 노이즈 3% + 시작 횡위치 ±5cm 지터.

보상 6항 중 3항(접근 shaping·행동 평활·타겟 주시)과 게이트 분리는 초기 설계가 아니라
**학습 진단을 통해 추가·수정된 것**이다 — 그 과정이 §5다.

## 4. 구현

### 4.1. 시뮬 환경 보강 (제조사 시뮬 → RL 환경)

제조사 monorepo(`src/simulations/`)는 수정하지 않고 우리 패키지로 보강했다:

- **온실 world** (`greenhouse_sim`): Gazebo Classic용 절차 생성기를 Ignition SDF로 이식,
  로봇 1.83× 스케일 포크, 카메라를 RGB-D `rgbd_camera`로 교체, LiDAR를 실물 MS200과
  동일한 360°로 확장.
- **이동 타겟**: velocity-control 플러그인이 Fortress에서 link gravity off를 지원하지 않아
  원기둥이 넘어지는 문제로 폐기 → static 모델을 **20Hz set_pose 키네마틱 구동**으로 구현.
- **타겟 특징 토픽** (`/target/features`): 로봇·작업자 ground truth에서 마커 특징을 역산하고
  가우시안 노이즈 3%를 주입해 발행. 화각 + **작물 벽 차폐(occlusion) 판정** 포함.
- **환경 리셋 경로**: `/world/.../set_pose` 서비스 브리지로 로봇 텔레포트 + 작업자 초기화
  (WorldControl 전체 리셋은 sim time까지 리셋되어 불채택).

### 4.2. rosorin_rl 패키지 (Gym ⇄ ROS 2 브리지)

데이터 흐름 다이어그램·코드 읽는 순서는 `rl_code_guide.md` §1. 파일별 역할:

| 파일 | 역할 |
|---|---|
| `follow_env.py` | 커스텀 Gymnasium Env — 내부 rclpy 노드로 토픽 구독/발행, step은 sim time 0.1s 주기 |
| `obs_pipeline.py` | 센서 → 16차원 정제(퍼센타일·EMA 필터, 뎁스범퍼 바닥 차감) → 3프레임 스택 48D |
| `reward.py` | 보상 6항 + 게이트 전환 + 종료 5조건 |
| `geometry_utils.py` | 핀홀 역투영(GT→마커 특징), 차폐 판정(작물 줄 AABB) |
| `target_controller_node.py` | 작업자 걸음 (전략 패턴 — 현재 `ConstantWalk`(정속 왕복)만 구현) |
| `target_feature_node.py` | `/target/features` 발행 — **Sim-to-Real 시 유일한 교체 지점** (§7) |
| `train_sac.py` | SB3 학습 드라이버 — `--algo sac/ppo`, 체크포인트+리플레이버퍼 저장, `--resume`/`--warm-start` |
| `eval_policy.py` | deterministic 평가 — 3지표 + 스텝별 CSV 덤프 |
| `callbacks.py` | TensorBoard 진단 지표 (종료사유 분포·보상 분해·행동 통계) |
| `config/rl_params.yaml` | 모든 튜너블의 단일 출처 (계수 의미·튜닝 가이드는 `rl_code_guide.md` §3) |

### 4.3. 시뮬 보정 (검증 중 실측으로 발견·수정)

상세는 `rl_code_guide.md` §5. 보고서에 쓸 만한 대표 3건:

1. **매카넘 y축 반전** — 벤더 MecanumDrive 플러그인이 `linear.y`를 REP-103과 반대로 해석.
   env가 발행 직전 부호를 보정한다 (실물 포팅 시 이 보정을 제거 — 시뮬/실물의 명확한 분리점).
2. **뎁스범퍼 바닥 차감** — 수평 카메라 하반 ROI에 바닥이 항상 보여 min이 상수가 되는 문제를
   행별 기대 바닥 깊이 차감으로 해결.
3. **직사각형 풋프린트 충돌 판정** — 단일 반경 임계로는 직사각형 로봇의 전/후면 접촉이
   미감지되는 버그를 빔 각도별 풋프린트 반경 차감(`env_margin`) 방식으로 교체, 4방향 실측 검증.

## 5. 학습 실험 이력 (조정 1~7차 연대기)

> 각 조정의 **설계적 근거**(수식·계수가 왜 그 값인가)는 `rl_design/rl_reward_function.md`,
> **운영 관점**(이 증상이면 이 값을)은 `rl_code_guide.md` §3. 여기서는 시간 순서의
> 진단 → 처방 → 결과를 기록한다.

### 5.0. 런 요약

| 런 | 스텝 | 알고리즘 | 적용 조정 | 타겟 속도 | 핵심 결과 |
|---|---|---|---|---|---|
| (초기 시도) | ~14k·~18k 중단 | SAC | 1차 설계 → 2·4차 진단 | 0.25→0.05 | 정체·배회 진단으로 보상 재균형 |
| sac_1 | 198k (762 ep) | SAC | ~4차 | 0.05 m/s | 학습 수렴 성공, 평가에서 잔존 결함 3종 발견 |
| sac_2 | 300k | SAC | 5~6차 | 0.05 m/s | 성공률 급상승(스냅샷 76%), 입구·편향 결함 잔존 |
| sac_3 | _진행 중 (150k+)_ | SAC | 7차 | 0.1 m/s (커리큘럼 1단계) | _TBD (학습 완료 후 갱신)_ |
| ppo_1 | _예정_ | PPO | 7차 (sac_3와 동일 조건) | 0.1 m/s | _예정 — SAC 대비 베이스라인_ |

### 5.1. 초기 시도와 sac_1 — 보상 신호가 "닿는지"부터 (1~4차)

- **1차(초기 설계) → 14k 정체**: 첫 200k 학습 시도가 14k 스텝에서 정체 (ep_len ~50 고정 =
  타겟 이탈 반복). 진단 — 가우시안 추종 보상은 원거리에서 기울기가 소실되고, 환경 충돌 -100이
  과도한 진입 공포를 만들었다. → **2차 조정**: w1↑·w2↓, 접근 shaping(`k_approach`) 신규,
  env_collision -100→-30, 타겟 속도 0.25→0.05(커리큘럼 0단계), 스폰 위치를 통로 입구로 확정.
  이때 진단 인프라(종료사유 분포·보상 분해·행동 통계 TB 로깅)도 구축 — 이후 모든 조정의 기반.
- **3차 — 충돌 감지 버그**: 단일 반경 임계로는 직사각형 로봇의 전/후면 접촉이 미감지됨을
  사용자가 GUI 관찰로 발견 → 풋프린트 마진 판정으로 교체 (§4.3-3).
- **4차 — "벽 투시"와 입구 배회**: 가시성이 화각만 검사해 작물 벽 너머 타겟도 visible=1이던
  버그(GUI 관찰로 발견)를 차폐 판정으로 수정. 18k 진단(d_t 1.4m 수렴·통로 진입률 급락 =
  입구에서 배회하는 국소최적)으로 α를 3.0으로 복원.
- **sac_1 완주 (198k/762 에피소드)**: 학습 곡선 자체는 정상 — lost율 0.67→0, 705번째
  에피소드에서 첫 성공 후 마지막 50 에피소드 성공률 96%. 그러나 **deterministic 평가**에서
  "입구 망설임 → 잔떨림 → 벽 충돌" 관찰, 최근 100 에피소드 충돌률 47%.

### 5.2. sac_2 — 평가 기반 정밀 진단 (5~6차)

sac_1 평가 CSV(스텝별 보상 분해·raw 센서 거리)로 원인을 특정:

- **5차**: ① `r_safety`가 학습 내내 0 (TB 실측) — 임계값이 LiDAR raw 기준으로 풋프린트보다
  안쪽이라 **물리적으로 도달 불가능**했다. 벽 회피 신호가 희소한 터미널 페널티뿐이었던 것
  → front 0.3→0.45m, side 0.05→0.32m (평가 CSV 실측 보정). ② 통로 모드 하드 스위치가
  입구에서 보상 불연속 점프를 만들어 진입 망설임 유발 → 선형 램프 전환. ③ 잔떨림에 비용이
  없음 → 행동 변화 페널티 `R_smooth` 신규.
- **6차** (재학습 전 보상 설계 감사): ① "타겟 바라보기"의 유일한 신호가 lost 절벽(-50)뿐
  → dense 선행 신호 `R_gaze` 신규. ② 입구 배회 보상 압착 — α 3.0→5.0 (sac_1에서 10회 중
  7회가 진입 전 ~11초 배회 실측). ③ 거리 유지 품질은 streak 보너스 대신(비마르코프라 기각)
  평가 지표(밴드 점유율·d_t σ)로 측정하기로 결정.
- **sac_2 결과 (300k, 보상이 바뀌었으므로 처음부터)**: 39% 시점 스냅샷 — 최근 50 에피소드
  성공률 76%, 통로 진입률 0.58→0.94, 행동 사용량이 탐색적 요동에서 미세 보정 제어로 전환
  (전체 지표는 `rl_code_guide.md` §4 실측 표). 잔존 결함 2종: **통로 진입 순간의 급격한
  자세 탐색**과 **통로 내 우편향 주행**.

### 5.3. sac_3 — 기하학적 재진단과 커리큘럼 1단계 (7차)

- **입구 관문의 기하학적 재진단**: 5차 램프는 측면 거리가 입구에서 서서히 줄어든다고
  가정했지만, 실제로는 잎벽이 측면 LiDAR 창에 들어오는 순간 ~2.1m→~0.4m **불연속 점프**.
  전환 구간에서 좌우 비대칭이 상한까지 포화 → 진입점에 0.25/step의 구조적 벌점("입구 관문")이
  형성돼 있었다. → 중앙 유지 항을 **먼 쪽 벽 기준 독립 게이트**(`center_gate`)로 분리 —
  양쪽 벽이 모두 가까울 때(진짜 통로 내부)만 활성.
- **우편향**: 치우침 비용이 추종 보상 대비 무시 가능 + 고정 스폰 궤적 고착이 원인
  → ζ 0.5→1.0 + 시작 횡위치 ±5cm 지터.
- **커리큘럼 1단계 진입**: 타겟 속도 0.05→0.1 m/s, max_steps 1200→1500 (한 에피소드 =
  통로 완전 왕복). 근거 실측 — sac_2를 0.1 m/s 환경에서 제로샷 평가하니 **5회 전부 84스텝
  이내 붕괴** (target_collision ×4: 과압박 평형 d_t≈0.42m, env_collision ×1: 입구 진동;
  `rl_logs/eval_sac2_baseline_v7env.csv`·`eval_sac2_nojitter_ab.csv`). 속도 상향은 재학습
  없이 불가함을 확인. 보상이 바뀌었으므로 리플레이 버퍼는 버리고 정책 가중치만 이식하는
  `--warm-start`로 sac_3 시작.
- **중간 경과 (2026-06-07, 150k+)**: monitor 꼬리 구간에서 1500스텝 완주(success, 리턴 ~2810)가
  우세. **최종 결과: _TBD (학습 완료 후 갱신)_**

## 7. 결론 및 향후 과제

**결론 (현시점):** 좁은 온실 통로에서의 근접 작업자 추종이라는 과제에 대해, 압축 특징
48차원 상태 + 6항 보상 설계로 SAC 정책 학습이 성립함을 확인했다. 특히 학습 곡선만으로는
보이지 않는 결함(도달 불가능한 페널티 임계, 보상 불연속이 만드는 구조적 망설임, 고정 스폰의
궤적 고착)을 **deterministic 평가 + 스텝별 보상 분해 CSV**로 특정하고 7차에 걸쳐 교정한
진단-처방 사이클이 본 프로젝트의 핵심 기여다. _최종 결론은 sac_3·PPO 완료 후 보강._

**향후 과제 (본 수업 범위 외):**

- **Sim-to-Real**: 설계상 교체 지점은 `target_feature_node.py` **하나뿐**이다 —
  `/target/features` 발행자를 실물 RGB-D 비전(AprilTag 검출 — 타겟 원기둥에 이미 공식
  tag36h11 패널 부착됨)으로 갈아끼우면 환경·정책 코드는 불변. 매카넘 y반전 보정 제거,
  실기 캘리브레이션 값 교체(`environment.md` 주의 절)가 동반 작업. Jetson Orin Nano 포팅
  및 실물 검증은 미수행.
- **시나리오 3~5** (후진 접근·지그재그·U턴): `target_controller_node.py`의
  `GaitStrategy` 상속 + 등록만으로 확장 가능한 구조는 마련됨 — `rl_design/rl_train_scenarios.md`.
  (시나리오 2 Stop&Go 는 `StopGo` 로 구현 완료 — 작물 단위 정지·이동.)
